## 4. Адаптация под PDF-экспорт

Презентация задумана под `@media print` / A4 landscape, но мелочи остались:

- [ ] **Проверить, что `aspect-ratio: 297/210` не ломает содержимое**
  в slide-content при печати. Поля `@page { size: A4 landscape; margin: 0 }` — есть,
  а вот контент рассчитан на 1120px max-width. На реальном A4 (297mm) при dpi 96
  это ~1123px — впритык. Добавить safety-padding.
- [ ] **Шрифты в PDF** — Inter подгружается с Google Fonts. При печати из браузера
  шрифт может не встроиться. Проверить: `print` → `pdf` и убедиться, что текст
  не превращается в sans-serif fallback. Альтернатива: подключить Inter локально.
- [ ] **Убрать любые `transition` / `animation` в print-режиме**.
  Пройтись по всем `*.css` и добавить `@media print { transition: none !important; }`.

## 6. Что НЕ нужно делать (anti-goals)

Чтобы не разрастаться, держать в&nbsp;фокусе:

- **Не** добавлять новые секции или фичи, которых не было в&nbsp;PDF.
- **Не** менять порядок PDF-нарратива: hook → инсайт → исследование →
  боли → гипотезы → цикл → продукт → маркетинг → открытые вопросы.
- **Не** подключать библиотеки графиков (recharts, d3, chart.js) — текущие
  SVG-компоненты проще и&nbsp;контролируемее.
- **Не** вводить состояние/роутинг/анимации между слайдами — это статичная
  презентация под PDF, а не SPA-приложение.
